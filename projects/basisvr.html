About BasisVR

BasisVR is an ambitious open-source project that democratizes social VR development. It provides a complete framework for building social VR platforms, handling everything from networking and avatar systems to audio processing and cross-platform VR support.

Built on Unity 6 with support for OpenXR, SteamVR, and Meta Quest, BasisVR enables developers to create their own social VR experiences without starting from scratch.

Technologies

BasisVR leverages a comprehensive stack of technologies for high-performance social VR.

Unity 6 + C#

Built on the latest Unity version with C# as the primary language, enabling rapid development and cross-platform deployment.

Unity Burst Compiler

High-performance LLVM-based compiler that translates IL/.NET bytecode to highly optimized native code, critical for real-time VR performance.

OpenXR + SteamVR

Cross-platform VR standards ensuring compatibility with major VR headsets including Quest, Index, Vive, and Windows Mixed Reality.

LiteNetLib

Lightweight reliable UDP networking library for low-latency multiplayer communication essential in social VR.

Opus Audio Codec

Industry-leading audio codec for high-quality, low-latency voice communication between users.

Steam Audio

Spatial audio solution providing realistic 3D sound propagation, occlusion, and HRTF rendering for immersive experiences.

One Euro Filter

Signal processing algorithm for smooth motion filtering with minimal latency, crucial for responsive avatar movement.

IK Systems

Inverse Kinematics systems for realistic avatar body positioning based on tracked head and hand positions.

My Contributions

January 2025

Burst-Compiled One Euro Filter for Network Jitter Resolution

The Problem

Remote avatars in the framework experienced visible micro-jitters caused by network compression artifacts. When avatar position and rotation data is transmitted over the network, compression introduces small inaccuracies that manifest as stuttering movements, breaking immersion and making avatars appear unnatural.

The Solution

Implemented a Burst-compiled One Euro Filter to smooth incoming network data while maintaining responsiveness. The One Euro Filter is an adaptive low-pass filter that adjusts its cutoff frequency based on input velocity - applying heavy smoothing during slow movements (when jitter is most visible) and minimal filtering during fast movements (to preserve responsiveness).

Why Burst Compilation Matters

The filter runs on every networked avatar, every frame. In a social VR environment with dozens of users, this means hundreds of filter calculations per frame. By making the One Euro Filter Burst-compatible, the code is compiled to highly optimized SIMD-vectorized native code, achieving near-C++ performance while remaining in the managed C# ecosystem.

  • Eliminated visible micro-jitters across all networked avatars
  • Achieved Burst compilation compatibility for maximum performance
  • Tuned filter parameters for optimal balance between smoothness and latency
  • Switched from deltaTime to interpolation time for more consistent filtering

Avatar Driver Architecture Refactor

The Change

Refactored the avatar driver system by splitting rig logic into a dedicated LocalRigDriver separate from LocalAvatarDriver. This architectural improvement provides cleaner separation of concerns and better maintainability.

  • Split monolithic avatar driver into focused, single-responsibility components
  • Improved code organization for the avatar and rig systems
  • Fixed Steam integration issues as part of the refactor
  • Enhanced the BasisLocalAvatarDriver for better extensibility
